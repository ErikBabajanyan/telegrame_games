§1. Architecture Overview

The frontend is a single-page React application running inside Telegram's Mini App WebView. It communicates with the backend REST API, handles real-time game state via WebSockets, and manages TON wallet connections through TON Connect 2.0.

§3. State Management (Zustand)

Four distinct Zustand stores handle different domains of state. Stores are persisted to sessionStorage where appropriate.

Mines Game Store

// stores/useMinesStore.ts
interface MinesState {
  // Active game session
  gameId: string | null
  status: 'idle' | 'active' | 'won' | 'lost'
  mineCount: 1 | 2 | 3 | 5 | 10 | 24
  betAmount: number          // in nanoTON
  cells: CellState[]         // 25 cells, indices 0-24
  revealedCount: number
  currentMultiplier: number
  currentPayout: number     // betAmount * multiplier
  serverSeedHash: string    // shown before game starts
  clientSeed: string
  nonce: number

  // Actions
  startGame: (bet: number, mines: number) => Promise<void>
  revealCell: (index: number) => Promise<void>
  cashOut: () => Promise<void>
  reset: () => void
}

type CellState = {
  index: number
  status: 'hidden' | 'gem' | 'mine'
  revealedAt: number | null  // timestamp
}

Wallet Store

interface WalletState {
  isConnected: boolean
  address: string | null         // TON wallet address
  internalBalance: number       // in nanoTON (casino balance)
  tonBalance: number            // on-chain TON balance
  walletName: string | null     // "Tonkeeper", "TON Space", etc.
  connect: () => Promise<void>
  disconnect: () => void
  refreshBalance: () => Promise<void>
  deposit: (amount: number) => Promise<void>
  withdraw: (amount: number) => Promise<void>
}

§4. Mines Game Flow

StepUser ActionFrontend ActionAPI Call
1Set bet amount + mine countUpdate store: betAmount, mineCount—
2Tap "BET"Validate balance ≥ betAmount; call startGame()POST /api/games/mines/start
3—Receive gameId, serverSeedHash; render 25 hidden cellsResponse: {gameId, serverSeedHash}
4Tap a cellMark cell as loading; call revealCell(index)POST /api/games/mines/reveal
5a—Cell is GEM → animate, update multiplier, show cashoutResponse: {result: 'gem', multiplier}
5b—Cell is MINE → shake, reveal all mines, game over modalResponse: {result: 'mine', minePositions}
6Tap "Cash Out"Show confirmation; call cashOut()POST /api/games/mines/cashout
7—Confetti, update balance, reveal server seed for verificationResponse: {payout, serverSeed}

§5. TON Connect Integration

// hooks/useTonConnect.ts
import { TonConnectUI } from '@tonconnect/ui-react'

const tonConnectUI = new TonConnectUI({
  manifestUrl: 'https://toncasino.app/tonconnect-manifest.json',
  actionsConfiguration: {
    twaReturnUrl: 'https://t.me/TonCasinoBot/app'  // Deep link back to TG
  }
})

// Deposit: send TON to escrow contract
async function sendDeposit(amountNano: bigint) {
  await tonConnectUI.sendTransaction({
    validUntil: Math.floor(Date.now() / 1000) + 600,
    messages: [{
      address: ESCROW_CONTRACT_ADDRESS,
      amount: amountNano.toString(),
      payload: buildDepositPayload(userId)  // BOC with userId
    }]
  })
}

§6. WebSocket (Real-time Events)

A persistent WebSocket connection carries game events and balance updates, avoiding polling.

EventDirectionPayload
game:cell_revealedServer → Client{cellIndex, result, multiplier, payout}
game:overServer → Client{reason, minePositions, serverSeed}
wallet:balance_updatedServer → Client{balance: nanoTON}
wallet:deposit_confirmedServer → Client{txHash, amount}
pingClient → Server{ts}

🔒 Security note: The Telegram initData string (signed by Telegram) is sent as a Bearer token on every API request. The backend verifies the HMAC-SHA256 signature against the bot token. This prevents forged user identities without requiring separate login.
