§1. Game Overview

HiLo is a card prediction game. A card is shown, and the player guesses whether the next card will be Higher or Lower. Each correct guess increases the multiplier. The player can cash out at any time.

🎯 Core mechanic: A standard 52-card deck. Player sees the current card and predicts the next one. Probability-based multipliers — the more obvious the guess, the lower the payout. Equal cards count as a loss.

How to Play

  • Player sets bet amount and taps "BET"
  • First card is dealt face-up
  • Player taps "HIGHER" or "LOWER" to predict the next card
  • Next card is revealed — if correct, multiplier increases
  • Player can "CASH OUT" at any time after 1 correct guess
  • If wrong (or equal), round is lost and bet is forfeited
  • Optional: "SKIP" — skip the current card (costs a small multiplier penalty)

§3. Mathematical Model

Card Values

Cards are valued A=1, 2-10 face value, J=11, Q=12, K=13. Each value has 4 suits (4 cards per value in a 52-card deck).

Probability Calculation

For a current card with value V (1–13):

GuessFormulaDescription
HighercardsAbove / (totalRemaining - cardsEqual)Cards with value > V
LowercardsBelow / (totalRemaining - cardsEqual)Cards with value < V
Equal (loss)cardsEqual / totalRemainingCards with value = V

Multiplier Calculation

// Per-guess multiplier (applied cumulatively)
guessMultiplier(probability) = (1 / probability) × (1 - HOUSE_EDGE)
guessMultiplier(probability) = (1 / probability) × 0.975

// Total game multiplier after N correct guesses
totalMultiplier = guess1_mult × guess2_mult × ... × guessN_mult

// Payout
payout = betAmount × totalMultiplier

💡 Dynamic odds: Unlike Mines, each round has different multipliers based on the current card. A "7" is the hardest — equal probability either way, so both HI/LO pay ~2x. Edge cards (A, K) give near-certain guesses with low payout.

Skip Mechanic

Player can skip the current card. A new card is dealt without affecting the multiplier. Cost: the skip itself takes a round — so if the next card is bad, you've burned your position. Max 3 skips per game. Skip does NOT require a guess.

§4. Per-Round RNG with Luck Factor

Same model as Mines — no pre-generated deck. Each card is drawn independently.

// hilo.engine.ts — draw next card
function drawNextCard(
  currentValue: number,
  guess: 'higher' | 'lower',
  luckFactor: number
): { value: number; suit: string; isCorrect: boolean } {
  // 1. Calculate base probability of winning this guess
  const winProb = calculateWinProbability(currentValue, guess)

  // 2. Adjust with luck factor
  const adjustedProb = clamp(winProb / luckFactor, 0.02, 0.98)

  // 3. Roll
  const isCorrect = cryptoRandom() < adjustedProb

  // 4. Generate a card that matches the result
  const card = generateCardForResult(currentValue, guess, isCorrect)

  return card
}

⚠️ Luck factor inversion: Unlike Mines where luckFactor > 1 makes mines MORE likely, in HiLo we divide the win probability by the luck factor. Higher luck factor = lower effective win chance. Same Redis key luck:{userId}, same hourly worker.
