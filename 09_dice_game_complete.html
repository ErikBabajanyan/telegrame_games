§1. Game Overview

Dice is a single-roll prediction game. The player chooses a target number (1–98) and a direction (Roll Under or Roll Over). A random number 1–100 is generated. If the result satisfies the condition, the player wins.

🎯 Core mechanic: Player controls their own risk. A low target on "Roll Under" = easy to win, low payout. A high target = hard to win, high payout. The player is effectively setting their own multiplier with every bet.

How to Play

  • Player sets bet amount
  • Player selects target number (2–98) using a slider
  • Player chooses "Roll Under" or "Roll Over"
  • Win chance and multiplier update in real-time as target changes
  • Player taps "ROLL" — single random number 1–100 is generated
  • If result satisfies condition → WIN (bet × multiplier credited)
  • If result doesn't satisfy → LOSS (bet forfeited)
  • Instant result — no multi-round, no cashout needed

Key Difference from Mines/HiLo

⚡ Single-roll game: Unlike Mines and HiLo which are multi-round games with cashout mechanics, Dice is a single bet → single result game. No game session to resume. Each roll is independent. This makes it the fastest game in the casino.

§3. Mathematical Model

Win Probability

// Roll Under: win if result < target
winProbability(target, 'under') = (target - 1) / 100

// Roll Over: win if result > target
winProbability(target, 'over') = (100 - target) / 100

// Examples:
// Under 50 → 49/100 = 49% win chance
// Over 50  → 50/100 = 50% win chance
// Under 95 → 94/100 = 94% win chance (easy, low mult)
// Under 5  → 4/100  = 4% win chance (hard, high mult)

Multiplier Calculation

multiplier = (1 / winProbability) × (1 - HOUSE_EDGE)
multiplier = (1 / winProbability) × 0.985

// Payout
payout = betAmount × multiplier

§4. Per-Roll RNG with Luck Factor

function rollDice(
  target: number,
  mode: 'under' | 'over',
  luckFactor: number
): { result: number; win: boolean } {
  // 1. Calculate base win probability
  const baseProb = mode === 'under'
    ? (target - 1) / 100
    : (100 - target) / 100

  // 2. Adjust with luck factor
  const adjustedProb = clamp(baseProb / luckFactor, 0.01, 0.99)

  // 3. Roll win/loss
  const isWin = cryptoRandom() < adjustedProb

  // 4. Generate a result number that matches
  const result = generateResultForOutcome(target, mode, isWin)

  return { result, win: isWin }
}

The result number is generated AFTER determining win/loss — ensuring the displayed number is consistent with the outcome. Same luck:{userId} Redis key, same hourly worker.

Roll Flow (single request)

StepAction
1Validate: betAmount (0.01–500 TON), target (2–98), mode (under|over)
2Deduct bet atomically from Redis balance
3Get user's luck factor from Redis
4Calculate win probability, run RNG roll
